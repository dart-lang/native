# Plan: Migrate `java.lang` and `java.nio` to `jnigen` Generated Bindings

This plan outlines the migration of hand-written JNI bindings in `package:jni` to bindings generated by `jnigen`. This will resolve naming conflicts (e.g., with `BigInteger`) and provide a more consistent API across the package.

## Overview

- **Target Package**: `pkgs/jni`
- **Goal**: Replace hand-written extension types in `pkgs/jni/lib/src/lang/` and `pkgs/jni/lib/src/nio/` with `jnigen` generated bindings in `pkgs/jni/lib/src/core_bindings.dart`.
- **Strategy**: 
    1. Update `pkgs/jni/tool/generate_jni_bindings.dart` to include the target class.
    2. Generate the bindings.
    3. Replace the hand-written extension type with extension methods that provide "Dart-ified" convenience methods (e.g., `toDartInt`).
    4. Fix all resulting analysis errors and broken tests. Iterate quickly by running jni fast checks.
    5. Ensure jni and jnigen full checks pass.

## Detailed API Changes

### `JNumber` (and subclasses)
- **OLD**: `int intValue({bool releaseOriginal = false})`
- **NEW (Generated)**: `int intValue()`
- **NEW (Extension)**: `int toDartInt({bool releaseOriginal = false})`
- *Repeat for `long`, `float`, `double`, `byte`, `short`.*

### `JBoolean`
- **OLD**: `bool booleanValue({bool releaseOriginal = false})`
- **NEW (Generated)**: `bool booleanValue()`
- **NEW (Extension)**: `bool toDartBoolean({bool releaseOriginal = false})`

### `JCharacter`
- **OLD**: `int charValue({bool releaseOriginal = false})`
- **NEW (Generated)**: `int charValue()`
- **NEW (Extension)**: `int toDartChar({bool releaseOriginal = false})`

### `JString`
- **OLD**: `String toDartString({bool releaseOriginal = false})`
- **NEW (Generated)**: `JString` methods like `substring`, `indexOf`, etc.
- **NEW (Extension)**: `toDartString` (Keep current implementation as it's optimized).

### `JByteBuffer`
- **OLD**: `Uint8List asUint8List({bool releaseOriginal = false})`
- **NEW (Extension)**: `asUint8List` (Keep current implementation).
